type: feedback
---

Every skill and agent must have a description section with a changelog table in README.md.

**Why:** The README is the central documentation for this shared repo. Without it, users have to read SKILL.md files to understand what's available.

**How to apply:** When creating a new skill or agent, add a section to README.md with: what it does, usage, key rules, and a changelog table. When modifying an existing skill or agent, add a new row to its changelog table in the README. Do this as part of the same commit — don't defer it.
